# Firebase 인증 도메인 설정 가이드

## 문제 해결

대문(랜딩 페이지)에서 로그인이 안 되는 경우, Firebase 인증 도메인 설정이 필요할 수 있습니다.

## 해결 방법

### 방법 1: localhost로 접속 (가장 간단)

브라우저에서 `http://localhost:8080` 또는 `http://127.0.0.1:8080`으로 접속하세요.

Firebase는 기본적으로 localhost와 127.0.0.1을 자동으로 허용합니다.

### 방법 2: 게스트 모드 사용 (로그인 불필요)

1. 랜딩 페이지에서 "로그인 없이 둘러보기 (게스트)" 버튼 클릭
2. 게스트 모드로 사용 가능 (데이터는 로컬에 저장)

### 방법 3: Firebase 콘솔에 도메인 추가

만약 IP 주소나 특정 호스트명으로 접속한다면:

1. [Firebase 콘솔](https://console.firebase.google.com/) 접속
2. 프로젝트 선택: `mealog-r0`
3. Authentication > Settings (톱니바퀴 아이콘)
4. "승인된 도메인" 섹션으로 스크롤
5. "도메인 추가" 클릭
6. 현재 접속 주소 입력:
   - IP 주소: `192.168.x.x` (서버 시작 시 표시되는 주소)
   - 또는 호스트명
7. 저장

## 권장 사항

개발/테스트 시에는:
- `http://localhost:8080` 사용 (추가 설정 불필요)
- 또는 게스트 모드 사용

## 참고

- Firebase는 기본적으로 `localhost`, `127.0.0.1`을 허용합니다
- IP 주소로 접속할 때만 도메인 추가가 필요합니다
- 게스트 모드는 항상 사용 가능합니다
